## Fix 1 — Heading colors must write to docx at XML run level

Current issue: H1/H2/H3/H4 colors set via style-level don't apply to actual runs in the output docx. Fix by applying color directly to each run's XML after setting paragraph style:

```python
from docx.oxml.ns import qn
from docx.oxml import OxmlElement
from docx.shared import RGBColor, Pt

def apply_run_color(run, hex_color):
    """Force color onto run via XML — bypasses style inheritance"""
    hex_clean = hex_color.lstrip('#').upper()
    rPr = run._r.get_or_add_rPr()
    # Remove existing color elements first
    for existing in rPr.findall(qn('w:color')):
        rPr.remove(existing)
    color_el = OxmlElement('w:color')
    color_el.set(qn('w:val'), hex_clean)
    rPr.append(color_el)

def apply_run_size(run, size_pt):
    run.font.size = Pt(size_pt)
    rPr = run._r.get_or_add_rPr()
    for existing in rPr.findall(qn('w:sz')):
        rPr.remove(existing)
    for existing in rPr.findall(qn('w:szCs')):
        rPr.remove(existing)
    sz = OxmlElement('w:sz')
    sz.set(qn('w:val'), str(int(size_pt * 2)))
    szCs = OxmlElement('w:szCs')
    szCs.set(qn('w:val'), str(int(size_pt * 2)))
    rPr.append(sz)
    rPr.append(szCs)
```

Apply `apply_run_color` and `apply_run_size` to every run in every heading paragraph — both in the formatter and the notebook converter. Also apply to body text runs for body font color.

## Fix 3 — Inline code rendering in markdown cells

When parsing markdown cells, inline code (text wrapped in single backticks like `` `variable_name` ``) must render as a Word run with:
- Font: Courier New
- Font size: same as surrounding body text (10pt)
- Background shading: `#E8E8E8` (light gray)

Implementation in `nb_converter.py`:

```python
def add_inline_code_run(paragraph, text):
    run = paragraph.add_run(text)
    run.font.name = 'Courier New'
    run.font.size = Pt(10)
    # Apply character shading via XML
    rPr = run._r.get_or_add_rPr()
    rShd = OxmlElement('w:shd')
    rShd.set(qn('w:val'), 'clear')
    rShd.set(qn('w:color'), 'auto')
    rShd.set(qn('w:fill'), 'E8E8E8')
    rPr.append(rShd)
```

When parsing inline markdown content, detect backtick-wrapped segments and call `add_inline_code_run` for those segments, `paragraph.add_run(text)` for normal segments.

## Fix 4 — Markdown tables → Word tables

When a markdown cell contains a pipe table like:

```
| Method | Samples/Update | Speed |
|--------|---------------|-------|
| Batch GD | All (1000) | Slow |
| SGD | 1 | Fast |
```

Parse it and render as a Word table with:
- First row: bold text, background fill `#D9D9D9`
- Alternating data rows: white / `#F7F7F7`
- All cells: thin single borders
- Font: TNR 11pt

```python
def render_markdown_table(doc, lines):
    # lines = list of pipe-separated rows, skip separator row (contains ---)
    rows = [l for l in lines if not re.match(r'^\|[-| :]+\|$', l.strip())]
    if not rows:
        return
    parsed = [[cell.strip() for cell in row.strip('|').split('|')] for row in rows]
    col_count = max(len(r) for r in parsed)
    table = doc.add_table(rows=len(parsed), cols=col_count)
    table.style = 'Table Grid'
    for i, row_data in enumerate(parsed):
        for j, cell_text in enumerate(row_data):
            cell = table.cell(i, j)
            cell.text = cell_text
            run = cell.paragraphs[0].runs[0] if cell.paragraphs[0].runs else cell.paragraphs[0].add_run(cell_text)
            run.font.name = 'Times New Roman'
            run.font.size = Pt(11)
            if i == 0:
                run.bold = True
                set_cell_shading(cell, 'D9D9D9')
            elif i % 2 == 0:
                set_cell_shading(cell, 'F7F7F7')
```

Detect markdown tables in cell content before passing to mistletoe — if a block of lines starts with `|` and contains a separator row, route to `render_markdown_table` instead.

## Fix 5 — Magic command handling in code cells

Jupyter magic commands and shell commands must not be passed to the Python tokenizer (they break syntax coloring and can cause crashes).

Before tokenizing a code cell, preprocess lines:

```python
def preprocess_code_lines(source_lines):
    processed = []
    for line in source_lines:
        stripped = line.strip()
        if stripped.startswith('%%'):
            # Cell magic — treat entire remaining cell as plain text
            return [(line, 'magic') for line in source_lines]
        elif stripped.startswith('%'):
            # Line magic — render as comment-colored line
            processed.append((line, 'magic'))
        elif stripped.startswith('!'):
            # Shell command — render as comment-colored line
            processed.append((line, 'shell'))
        else:
            processed.append((line, 'code'))
    return processed
```

Render `magic` and `shell` typed lines in italic gray (`#666666`) Courier New — visually distinct from normal code, not passed to tokenizer.

```python
import matplotlib.pyplot as plt
import io, base64

def latex_to_image(latex_str):
    fig, ax = plt.subplots(figsize=(4, 0.8))
    ax.text(0.5, 0.5, f'${latex_str}$', size=14, ha='center', va='center')
    ax.axis('off')
    buf = io.BytesIO()
    fig.savefig(buf, format='png', bbox_inches='tight', dpi=150)
    plt.close(fig)
    buf.seek(0)
    return buf
```
