**Bug 2 — Enrollment number gets extra character prepended in footer:**
In `nb_converter.py` (or wherever footer is built), the footer paragraph likely has a leftover run or page number field that is concatenating with the enrollment text. Fix:

```python
def set_footer(section, left_text, right_text, include_page_num=False):
    footer = section.footer
    para = footer.paragraphs[0]
    para.clear()  # ← CRITICAL: clear ALL existing runs first
    
    # Tab stop at right margin
    pPr = para._p.get_or_add_pPr()
    tabs = OxmlElement('w:tabs')
    tab = OxmlElement('w:tab')
    tab.set(qn('w:val'), 'right')
    tab.set(qn('w:pos'), '9360')
    tabs.append(tab)
    pPr.append(tabs)
    
    # Left: lab number
    run1 = para.add_run(left_text)
    run1.font.size = Pt(10)
    run1.font.name = 'Times New Roman'
    
    if include_page_num:
        para.add_run('\t')
        # center page number as separate paragraph or inline
        add_page_number(para)
        para.add_run('\t')
    else:
        para.add_run('\t')
    
    # Right: enrollment number — isolated run, nothing else appended
    run2 = para.add_run(right_text)
    run2.font.size = Pt(10)
    run2.font.name = 'Times New Roman'
```

Key: `para.clear()` before building — existing template footer likely has a page number field already embedded that bleeds into ur text.

**Bug 3 — Header subject not right-aligned:**
Same issue — tab stop position `9360` (in twentieths of a point = twips) assumes default margins. If margins differ, subject drifts to center. Use this calculation:

```python
from docx.shared import Inches
# Page width 8.27" (A4), left margin 1", right margin 1" = text width 6.27"
# 6.27 inches * 1440 twips/inch = 9029 twips
tab.set(qn('w:pos'), '9029')  # A4 with 1" margins
```

If using 2.54cm (1 inch) margins on A4: text width = 21cm - 5.08cm = 15.92cm = 9029 twips. Use `9029` not `9360`.
